after send
sheet dismisses on success · confirmation toast, 3s · success haptic fires on the send tap, not the toast · back arrow returns to the chat
backend, for the record. all three forms post to one gateway endpoint (POST /api/v1/feedback) with a kind of wrong_answer · bug · message. nestjs validates, stamps the verified user id + account email + app version, and forwards to a slack webhook — no database, no fastapi, kebi's brain never hears about it. only wrong_answer attaches the transcript turns with reasoning steps and tool results. rate-limited like the other gateway routes.
